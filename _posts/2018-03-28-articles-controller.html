Rails model is useless without a corresponding controller. We need controllers to connect the database with the views and perform "CRUD" actions.

Make sure you're in the right directory and type in your terminal:

rails g controller Articles

Rails has just generated an Articles controller with corresponding css and coffeescript files. Rails is really smart!

Ruby on rails has its own conventions and thoughts about performing the standard "CRUD" actions. Open the newly created Articles controller file ()

  • index
  • show
  • new
  • edit
  • create
  • update
  • destroy

With actions a controller is pretty much useless. Rails philosophy insists on having a seven actions for the "CRUD" functionality. Let's start with the first one: new.

Open "app/controllers/application_controller.rb" file and create a new function called "new":

  end
end
